# AI作業場

複数のAI(Claude / GPT / Gemini)に「役割」を与え、1つのチャット欄から使い分け・連携させる個人用デスクトップアプリ。
ローカルフォルダの資料から締め切りを自動検知するタスク管理機能も持つ。

正式な仕様は [docs/AI作業場_実装指示書.md](docs/AI作業場_実装指示書.md) を参照(本書はその実装版の概要)。

## 主な機能

| 分類 | 機能 |
|---|---|
| チャット | `@役割名` メンションで宛先カードを指定 / ストリーミング表示 / AI別の色付き外枠 |
| 役割カード | 名前・プロバイダ・モデル・システムプロンプト・参照資料・ツール許可・トリガーの単位でAIに役割を付与 |
| モデル切替 | カード恒久変更+「この送信のみ」の一時上書きの2段階(§4.2)。説明付きモデルピッカー |
| 共有バイブル | 全カードが参照する資料。API呼び出し時にシステムプロンプトへ注入(§4.3) |
| 保管庫 | AIの成果物を完成品として保存(タグ+全文検索、出所パンくず付き)。AIには読み込ませない |
| パイプライン | 複数カードの直列実行(執筆→矛盾チェック→校正 等)。確認質問での一時停止・再開対応(§4.4) |
| 会話分岐 | メッセージはツリー構造(`parent_id`+`active_leaf_id`)。枝切替・横並び比較・本線変更(§4.5) |
| 複数モデル比較 | 同一プロンプトを複数カードへ同時送信し横並び表示 |
| 確認質問 | 情報不足時にAIがJSONで質問を返し、選択肢UIで回答して続行(§4.16) |
| Web取得 | 許可カードのみ、ユーザーが明示したURLを取得(robots.txt尊重・重複取得なし §4.9) |
| 締め切り検知 | フォルダ定期スキャン→PDF/PPTX/DOCX抽出→AI検知→**確認UIを経て**タスク登録(§4.6) |
| 朝ブリーフィング | 1日の最初の起動時に秘書役がタスクを整理して表示(§4.8) |
| 生成スタジオ | 画像生成(OpenAI/Google)+ギャラリー+背景に設定(§4.14) |
| コスト | トークン使用量をAI別・日別に記録(グラフ)。単価入力で概算コスト・キャッシュ節約額を表示(§4.11) |
| 役割仕分けウィザード | やりたいことを書くと3社が同時に役割分担を提案→カード自動生成(§4.10) |
| テーマ・背景 | ライト/ダーク(CSS変数)+静止画/動画背景の3層レイヤー(§4.12) |

## 技術構成

- **Electron + React + Vite + TypeScript**(仕様書§2の許容に基づきTauriではなくElectronを採用)
- AI接続は各社公式SDK(`@anthropic-ai/sdk` / `openai` / `@google/genai`)、BYOK方式
- モデル一覧は各社APIから動的取得(ハードコードなし §3)
- プロンプトはキャッシュ効率を最優先に「①システム→②バイブル→③要約+履歴→④入力」の順序固定(§8.1)、古い履歴は軽量モデルで要約圧縮(§8.2)
- 文書抽出は純JS(`pdf-parse` / `adm-zip`)

## データの保存場所

- **ワークスペース**(ユーザーが選ぶフォルダ。クラウド同期フォルダ推奨): 役割カード・会話・共有バイブル・保管庫・パイプライン・タスク・使用量。会話は1件1ファイル、使用量は月別ファイル(同期衝突の最小化 §6)
- **PCローカル**(`userData`): APIキー(Electron `safeStorage` でOS暗号化。**同期フォルダには絶対に置かない** §9)、キャッシュ

## 開発

```bash
npm install        # 依存の導入
npm run dev        # 開発モード(Vite+Electron、ホットリロード)
npm start          # ビルドして起動
npm run dist       # 配布用ポータブルexeを release/ に生成
```

Windowsでダブルクリック起動する場合は `AI作業場を起動.bat`。

## 2台のPCで使う

1. ワークスペースをGoogle Drive等の同期フォルダ内に作る
2. もう1台では `release/AI作業場-portable.exe` をコピーして起動し、同じワークスペースフォルダを選ぶ
3. APIキーは各PCで初回に入力する(設計上、キーは同期されない)

## セキュリティ方針(§9)

1. APIキーは同期フォルダ・設定JSONに平文保存しない(OS暗号化のみ)
2. Web取得はユーザーが明示したURLのみ。自動クロールなし
3. 締め切り検知は必ずユーザー確認を挟んでからタスク登録
4. 外部送信はユーザー自身のキーによる各社公式APIへのリクエストのみ
