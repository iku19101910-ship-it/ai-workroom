## タスクA: アトミック書込+ゴミ箱(SHARED-010)

**目的**: 書き込み中のクラッシュ・強制終了でJSONが壊れるのを防ぐ。削除の誤操作を復元可能にする。

1. `electron/lib/fsutil.cjs` を新規作成:
   - `atomicWriteFile(filePath, data)`: `filePath + ".tmp"` に書き込み後 `fs.renameSync` で置換(Windowsで既存ファイルへのrenameは上書きされる。失敗時はtmpを削除して再throw)
   - 親ディレクトリの mkdirSync -p も内包する
2. 以下の直接 `fs.writeFileSync` をすべて `atomicWriteFile` に置換:
   - `workspace.cjs` の `writeJson` と `.md` 書き込み(saveSharedDoc / saveArtifact)
   - `config.cjs` / `keys.cjs` / `gen.cjs`(index.json。画像本体は新規作成なので対象外)/ `scanner.cjs`(scan_state)/ `briefing.cjs` / `webfetch.cjs`(キャッシュ)
3. **ゴミ箱**: `ワークスペース/.trash/` を導入。`workspace.cjs` の削除系
   (deleteConversation / deleteSharedDoc / deleteArtifact / deletePipeline / deleteRoleCard)と `gen.cjs` の deleteMedia は、
   unlink の代わりに `.trash/{ISO日時(コロンは-に置換)}_{元ファイル名}` へ移動する(索引からの除去は従来どおり)
4. アプリ起動時(main.cjs の ready 内、try/catch)に30日より古いゴミ箱ファイルを削除する掃除関数を呼ぶ
5. README.md に「削除したデータはワークスペースの `.trash` に30日間残る」旨を1行追記

**確認**: 会話を1件作って削除→`.trash` に移動していること。設定変更が正常に保存されること。

## タスクB: プロジェクト単位の管理(SHARED-011)

**目的**: 会話・成果物・タスク・パイプライン・共有バイブルを「プロジェクト」で束ね、文脈の混在を防ぐ。

### データ
1. `ワークスペース/projects.json`: `{ "projects": [{ "id": "prj_...", "name": "小説◯◯", "color": null, "created_at": ISO, "archived": false }] }`
2. `project_id: string | null` フィールドを以下に追加(既存データは undefined=未分類として扱い、**移行処理は不要**):
   - 会話(会話ファイル本体と conversations/index.json の両方)
   - 共有バイブル index の各doc / 保管庫 index の各item / tasks.json の各task / パイプライン定義
3. **役割カードはプロジェクトに属させない**(全プロジェクト共通の道具。変更禁止)
4. 現在選択中のプロジェクトは **PCローカル**の appConfig(`config.cjs`)に `currentProjectId` として保存(2台のPCで別々の作業をするため同期しない)

### メイン/IPC
- `projects:list` / `projects:save`(新規・改名)/ `projects:archive`(削除はせずarchived=true。所属データは未分類に戻さずそのまま)
- `app:setCurrentProject(id | null)` → appConfig 保存
- 既存の作成系IPCに project_id を通す: `convs:create(title, projectId)`、cards以外の save系(doc/artifact/task/pipeline)は引数オブジェクトの project_id をそのまま保存
- パイプライン実行(`pipeline.cjs` runPipeline)が作る会話にも、呼び出し時に渡された projectId を付与する(IPC `pipelines:run` に projectId 追加)

### UI
- **Header.tsx にプロジェクト切替ドロップダウン**(タイトルの右、テーマ切替の左)。項目: 「全体」+ 各プロジェクト + 「未分類」+ 区切り + 「+ 新規プロジェクト」「プロジェクト管理…」
  - 新規: 名前入力の小モーダル。管理: 一覧で改名・アーカイブできる小モーダル
- App.tsx で `currentProjectId: string | null | "uncategorized"` を state 管理し、各ビューに prop で渡す
- フィルタ規則: 「全体」=全表示 / プロジェクト選択中=その project_id のみ / 「未分類」= project_id が null・undefined のみ
- フィルタ適用対象: チャットの会話一覧、保管庫一覧、タスク一覧(検知候補は全表示のまま)、バイブル一覧、パイプライン一覧、ホームの各パネル
- **新規作成物**(会話・成果物・タスク・資料・パイプライン)は現在のプロジェクトを自動付与(「全体」「未分類」選択中は null)
- 各一覧行に所属プロジェクト名の小バッジ(「全体」表示時のみ。muted色)

**確認**: プロジェクト2つ作成→切替で一覧が絞られる→新規会話が自動所属→「全体」で両方見える。

## タスクC: 全文横断検索(SHARED-012)

**目的**: 会話・バイブル・保管庫・タスク・カード・パイプラインを1つの検索欄から探し、**その場に飛べる**ようにする。

### メイン側
1. `electron/lib/search.cjs` 新規: `searchAll(query)` — 小文字化した部分一致で走査:
   - 会話: タイトル+全メッセージ content(ヒットメッセージの id も返す)
   - 共有バイブル: タイトル+本文 / 保管庫: タイトル+タグ+本文 / タスク: タイトル / カード: 名前+システムプロンプト / パイプライン: 名前
   - 返り値: `{ type: "conversation"|"doc"|"artifact"|"task"|"card"|"pipeline", id, title, excerpt(ヒット位置の前後40字、\nは空白化), project_id, meta }`。conversation の meta は `{ message_id }`
   - 種類ごとに最大10件で打ち切り。空クエリは空配列
2. IPC `search:global(query)`

### UI
1. `src/components/GlobalSearch.tsx` 新規: コマンドパレット型オーバーレイ
   - Header の 🔍 ボタン、または **Ctrl+K**(App.tsx で keydown 監視)で開く。Escで閉じる
   - 入力は250msデバウンスで検索。結果は種類別グループ見出し(💬会話 / 📚資料 / 📦成果物 / 📋タスク / 🃏カード / ⛓パイプライン)
   - 各行: タイトル+抜粋(クエリ部分を `<mark>` 相当のアクセント色で強調)+プロジェクト名バッジ
   - ↑↓で選択、Enter/クリックで遷移して閉じる
2. **遷移の配線**(App.tsx に `openTarget` state を追加し、該当ビューへ切替+propで渡す):
   - conversation → chat ビューへ。`ChatView` に新prop `initialConversationId?` `focusMessageId?` を追加。会話を開き、該当メッセージへスクロール+2秒ほどアクセント色の枠で強調。**ヒットが本線外の枝にある場合は本線を切り替えず**、ツリーモーダルを開いてそのノードを選択状態にする
   - doc → cards ビュー(バイブルタブ)で該当資料の編集モーダルを開く(`CardsHubView` に `initialTab?` / `SharedMemoryView` に `initialDocId?`)
   - artifact → artifacts ビューで詳細モーダルを開く(`ArtifactsView` に `initialOpenId?`)
   - task / card / pipeline → 該当ビューへ切替のみ(モーダルまでは不要)
   - prop は「一度消費したら親が null に戻す」方式で再発火を防ぐ

**確認**: 会話内の単語で検索→チャットの該当メッセージへ飛んで強調される。保管庫本文ヒット→詳細が開く。

## タスクD: 自動引き継ぎメモ(SHARED-013)

**目的**: 会話ごとに「決定事項・未解決・次の作業」を軽量モデルで自動抽出し、続きの作業(別AI含む)へ渡す。

### データ・メイン側
1. 会話ファイルに `handoffs: [{ id, created_at, decisions: string[], open_issues: string[], next_steps: string[], model }]` を追加(配列末尾が最新)
2. `electron/lib/handoff.cjs` 新規:
   - `generateHandoff(conversationId)`: 本線メッセージ(mainlineForPrompt)を transcript 化し、`pickCheapModel`(§8.4。プロバイダは会話の最後のアシスタントのカード→無ければキー設定済みの先頭)で
     「次のJSONのみで出力: {"decisions":[],"open_issues":[],"next_steps":[]}(日本語、各最大5項目、無い場合は空配列)」と依頼。
     フェンス除去+JSON.parse の防御的パース(prompt.cjs の実装を参考)。成功時 handoffs へ追記し `recordUsage`(role_card_id はそのカードid)
   - `maybeAutoHandoff(conversationId)`: settings.handoff.auto が true かつ「最新handoff作成時刻より後の本線メッセージ数 >= settings.handoff.min_new_messages」のときだけ generate。実行中の会話idを Set で多重防止。失敗は握りつぶす(自動系のため)
3. settings.json 既定値に `handoff: { auto: true, min_new_messages: 6 }` を追加(DEFAULT_SETTINGS と getSettings の補完)
4. IPC: `handoff:generate(conversationId)` / `handoff:auto(conversationId)`(=maybeAutoHandoff、awaitさせない)

### UI
1. ChatView: 入力コントロール行に「📝 引き継ぎ」ボタン(会話未選択時disabled) → モーダル:
   - 最新メモを「✅ 決定事項 / ❓ 未解決 / ▶ 次の作業」の3セクションで表示(無ければ「まだありません」)
   - 「🔄 いま生成」ボタン(handoff:generate、生成中表示)。過去メモは `<details>` で時系列表示
2. **自動生成トリガ**: ChatView で会話を切り替える時と window の `beforeunload` 時に、直前の会話へ `handoff:auto` を投げる(fire-and-forget)
3. **「続きから」動線**: ホームの現在プロジェクトパネルに「最新の引き継ぎ」を表示し、「▶ この続きで新しい会話」ボタンを設置。押すと: chat ビューへ遷移→現在プロジェクト所属の新規会話を作成→入力欄に
   `【前回からの引き継ぎ】\n決定事項: …\n未解決: …\n次の作業: …\n\n` をプリセット(ユーザーが編集して送信する。自動送信はしない)。
   App.tsx の openTarget 機構(タスクC)を流用し、ChatView に `presetText?` prop を追加する
4. 設定画面: 「引き継ぎメモ」セクション(自動生成ON/OFF、しきい値往復数)

**確認**: 6往復以上の会話で別会話に切替→数秒後に handoffs が追記される。ホームから「続きから」→入力欄にメモが入る。
